# 规划设计院合同管理系统 - MVP内部体验与测试指南

欢迎参与本期合同管理系统的首航内部试用！此版本为 **最小可行性产品 (MVP)**，核心目标是**“验证从项目立项到资金安全回收的骨干业务连通性”**，排除花哨边缘功能，直击痛点。

---

## 一、系统试用说明与推荐操作流

### 🎯 1. 我们当前能做什么？

系统目前可一站式接管规划设计类业务的“确权—>拆解—>进账—>开票”全周转链，自动隔离权限，并提供上帝视角的金钱安全预警（防垫资、防欠票）。

### 👥 2. 测试角色与典型使用场景

我们在试用版中安置了三个“模拟门禁”，大家可分别登录换位体验：

* **账号 `admin` (密码 `123456`) - 系统超管/院级复核人**
  * **能做的事**：览遍全院所有台账、查看“实收开票漏斗” Dashboard 发呆、进入【系统设置】偷偷查看任何人在系统内的底细（操作审计留痕）。
* **账号 `manager` (密码 `123456`) - 规划一所所长**
  * **能做的事**：仅能查看“自己部门”所有组员的合同池和收据池，隔离了别部门的机密。
* **账号 `pm` (密码 `123456`) - 项目经理/业务员小李**
  * **能做的事**：系统中最苦最累的源头。负责登记录入自己的客户、给自己的项目去立项、去签发起草属于自己的合同。

### 🔄 3. 推荐大家试用的核心“连招”步骤

1. **新建底座**：用 `pm` 登录 -> [客户管理] 建个虚假甲方 -> [立项报备] 关联这个甲方立个项。
2. **生成合同**：在 [合同台账] -> 起草合同 -> 下拉选择刚建的项目。体会一下系统是如何把项目预算和部门自动继承锁定过来的。
3. **分配计划**：点开刚建好的合同详情。在下方【收款计划】里安排3期打款。**（恶意测试环节：故意把3期计划总和填得大于合同总额，看看防爆雷弹窗会不会跳出来咬你。）**
4. **回款与开票碰库**：在【实收款】录入一笔现金 -> 在【发票流水】虚开一笔发票金额。
5. **体验 Dashboard 报警**：退号换 `admin` 登录进主控台，看看主页右上角的“逾期催收池”和“垫税黑洞池”有没有把你刚造的那些荒唐数字给报警挂出来！

## 二、当前版本已知限制说明

在体验中，如果你点到某些地方觉得“不够爽”，请先息怒，这是我们在这个跑通周期的“策略性留白”：

### 🧱 1. 哪些能力是简化实现的？ (MVP特有)

* **登录体系**：你们现在用的是我手敲在系统里的假名，真正的企业微信扫码登录/单点登录在后头。
* **文件上传**：挂载档案现在是真的“扔”到了服务器的本地硬盘里供下载，没有接驳阿里的 OSS 云端和在线预览器。
* **操作员回显**：你在这个系统里的任何增删改都会被挂上“内部员工”或者“业务员小李”的名号。

### 🚧 2. 哪些地方压根还没做？

* **自动化审批流**：现在的合同状态是硬切的，还没接入“提交部门主管 -> 分管院领导 -> 财务审计 -> 法务盖章” 的冗长审批串联引擎。
* **变更补充协议**：目前主合同一经落定（特别是总额），不支持打“补丁协议”去追加金钱。

## 四、研发：下一阶段 (Sprint 6+) 演进蓝图

打完了这场 MVP 的攻坚战，如果顺利挺过大家的使用，我们的下一步高配弹药已经备好：

### 🔴 高能级优先突破

1. **真实身份基建互联**：干掉那三个 mock 的登录名，对接企业微信应用鉴权网关或局域网 AD 域，把整个系统的“树形部门组织架构”活化。
2. **多态大批量导出**：顺延台账导出的成功，快速在“发票池”和“实收款池”开放条件导出，让财务大姐笑逐颜开。
3. **合同主体变更支持**：引入《补充协议书》与《重大终止》的从属架构变更机制。

### 🟡 可排期的中期效能提升

1. **系统级定时飞信播报**：深夜跑批，每天早上 8 点把“今天有哪些合同该催款了”、“哪些老板的发票倒欠了”通过企微机器人直接砸给业务员。
2. **自定义系统数据字典配置**：把目前写死的“合同类型(规划/测绘/建筑)”、“发票税点”开放一个配置页面给行政小妹自己配。

### 🟢 长期护城河战役

1. **深度定制工作流流转 (BPMN)**：不让员工自己改“履约状态”，一切靠工作流表单流转审批驱动状态机。
2. **票据与合同的 AI OCR 查验**：财务拿着几百万的实体发票图片上传，机器自动校验发票核额合法性及票号去重。
